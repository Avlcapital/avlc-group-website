Presentation Pack

AVLC Group Website

Prepared for: Dmura

This pack is designed for two presentation days: a business-facing walkthrough and a technical or operational follow-up.

Corporate Website Admin Updates Email Forms Tender Documents

Executive Summary

The AVLC Group website is a modern multi-page corporate platform that presents the AVLC brand, showcases group companies, highlights patented products, publishes tender and update information, and receives customer inquiries directly through email-enabled forms.

The website also includes an authenticated admin area that allows internal users to create, edit, publish, and manage updates without editing source code directly.

Current Scope

Public Pages

  • Homepage
  • About AVLC Group
  • Company profile pages
  • Our Patented Products
  • Tender Openings
  • Updates
  • Contact

Admin Pages

  • Admin login
  • Updates publisher

Operational Features

  • Contact messages to company email
  • Asset recovery form submissions to company email
  • MongoDB-backed admin authentication
  • Local website assets and documents

Feature Highlights

Area What It Does Presentation Value
Homepage Introduces the group, presents the introduction section, and highlights the group companies. Strong first impression and clear positioning.
About Page Shows the group story, vision, values, directors, and partners. Builds trust and corporate credibility.
Companies Dedicated pages for AVLC subsidiaries accessed from a dropdown menu. Shows the group as a structured business ecosystem.
Patented Products Shows product cards with clickable popouts and detailed sections below. Interactive and easy to demonstrate live.
Tender Openings Organizes tender information by country with downloadable documents. Shows practical business use beyond marketing content.
Updates Public update timeline with admin publishing flow. Demonstrates ongoing content operations.
Contact Provides address, map, and message form routed to email. Shows lead capture and communication readiness.

Admin and Operations

Admin Authentication

Admins sign in with username and password. Access is protected with session cookies and backed by MongoDB session validation.

Updates Publishing

Admins can create, edit, publish, unpublish, and delete update entries that appear on the public Updates page.

Email Delivery

Contact and asset recovery submissions are routed through SMTP and delivered to designated company inboxes.

Technical Stack

Next.js App Router React TypeScript Tailwind CSS MongoDB Nodemailer

The implementation uses a server-rendered Next.js application for public pages, MongoDB for admin users and sessions, and API routes for form submissions and update publishing.

Architecture Snapshot

Layer Responsibility
Public UI Corporate content, company profiles, patented products, tenders, updates, and contact pages.
Admin UI Login and updates management.
API Routes Handle form submissions, login, logout, and update management.
Email Layer Sends contact and asset recovery submissions to company email.
Data Layer MongoDB stores admin users and session data. Update content supports admin-driven publishing.
Asset Layer Local images and downloadable documents are stored inside the project for better reliability during demos.

Thursday Presentation Plan

Recommended Focus

Use Thursday for business value, brand presence, and user journey.

  1. Start from the homepage and explain AVLC Group positioning.
  2. Show the Introduction section and the group company highlights.
  3. Open the About page and explain the corporate story, directors, and partners.
  4. Use the Our Companies dropdown to show the group structure.
  5. Open the Patented Products page and demonstrate one interactive product popout.
  6. End with Contact and explain how inquiries flow into company email.

Friday Presentation Plan

Recommended Focus

Use Friday for administration, operations, and implementation maturity.

  1. Show the Updates page and explain public communication value.
  2. Sign in to the admin side.
  3. Create or edit an update.
  4. Publish the update and show it reflected on the public page.
  5. Explain contact and asset recovery submission handling.
  6. Close with the stack, MongoDB-backed admin auth, and local asset strategy.

Presentation Checklist

  • Verify company dropdown and company pages open correctly.
  • Verify patent product popout works.
  • Verify tender documents open.
  • Verify contact form is configured in the target environment.
  • Verify admin login works in the deployment environment.
  • Verify update publishing works before the presentation session starts.

Closing Statement

The AVLC Group website now functions as a corporate presentation platform, a communication channel, a document delivery point, and an admin-managed public updates system. That makes it useful both as a brand site and as a practical operational tool.
